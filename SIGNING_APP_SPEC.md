# Спецификация AI-сервиса подписи PDF

## 1. Назначение

Нужно создать локальное веб-приложение, которое позволяет загрузить несколько PDF-документов, автоматически найти места для подписи и печати, вставить изображения подписи и печати, при необходимости добавить ФИО `Венедиктов Р.В.`, проверить результат в предпросмотре и выгрузить готовые PDF по отдельности или ZIP-архивом.

Первая версия приложения работает только с PDF. Поддержку DOCX нужно рассматривать как отдельный этап, потому что DOCX редактируется как документная структура, а PDF - как визуальная страница с координатами.

Ключевая идея: локальный OCR/CV-алгоритм находит кандидатов, а AI помогает понять смысл контекста и выбрать правильные места. AI не должен быть единственным механизмом принятия решения: при ошибке API или низкой уверенности сервис обязан продолжить работу локально и отправить спорные документы на ручную проверку.

## 2. Основной пользовательский сценарий

1. Пользователь открывает локальный веб-интерфейс.
2. Загружает один или несколько PDF-документов.
3. Выбирает галочками нужные действия:
   - поставить подпись;
   - поставить печать;
   - добавить `Венедиктов Р.В.`, если ФИО нет рядом с местом подписи;
   - использовать AI-анализ;
   - требовать ручное подтверждение перед экспортом.
4. Сервис анализирует документы и показывает найденные места подписи/печати.
5. Пользователь открывает предпросмотр каждого документа.
6. Пользователь может:
   - принять автоматическое размещение;
   - перетащить подпись;
   - перетащить печать;
   - изменить размер подписи или печати;
   - включить или выключить конкретную подпись/печать;
   - добавить место подписи вручную.
7. Пользователь экспортирует результат.
8. Сервис отдает обработанные PDF или ZIP-архив.

Исходные документы нельзя перезаписывать. Все результаты сохраняются как новые файлы.

## 3. Функциональные требования

### 3.1 Загрузка и обработка файлов

- Поддержать загрузку нескольких PDF за один раз.
- Для каждого файла создать отдельную задачу обработки.
- Показывать статус по каждому файлу:
  - загружен;
  - анализируется;
  - найдено место;
  - нужна ручная проверка;
  - готов к экспорту;
  - ошибка.
- Не блокировать весь пакет, если один документ обработался с ошибкой.
- Ограничить типы файлов на уровне UI и backend: принимать только PDF.
- Хранить загруженные файлы во временной рабочей папке.

### 3.2 Настройки пользователя

В интерфейсе должны быть чекбоксы:

- `Поставить подпись`
- `Поставить печать`
- `Добавить Венедиктов Р.В., если ФИО нет`
- `Использовать AI-анализ`
- `Требовать ручное подтверждение перед экспортом`

Настройки должны применяться как ко всему пакету, так и сохраняться в конкретной задаче документа. Если пользователь вручную отключил подпись или печать в одном документе, это решение должно сохраниться только для этого документа.

### 3.3 Предпросмотр и ручная корректировка

Предпросмотр обязателен. Даже при высоком confidence пользователь должен иметь возможность проверить результат до экспорта.

В предпросмотре нужно реализовать:

- отображение страниц PDF как изображений;
- наложение подписи, печати и ФИО поверх страницы;
- drag-and-drop для подписи и печати;
- изменение размера;
- удаление конкретного placement;
- добавление нового placement вручную;
- отображение confidence и причины автоматического выбора;
- предупреждение, если документ требует ручной проверки.

Координаты правок должны сохраняться в PDF-координатах, а не только в пикселях браузера. Это нужно, чтобы экспорт был стабильным при разных масштабах предпросмотра.

## 4. Изображения подписи и печати

Дефолтные ассеты:

- подпись: `/Users/rafailvv/Documents/подпись.png`
- печать: `/Users/rafailvv/Documents/Печать.png`

Требования:

- PNG-прозрачность должна сохраняться.
- Подпись вставляется поверх линии или чуть выше линии.
- Печать вставляется после подписи и должна частично захватывать подпись.
- Печать не должна полностью закрывать ФИО или важный печатный текст.
- Размер печати должен выглядеть реалистично: ориентир 35-45 мм по ширине/высоте.
- Размер подписи должен рассчитываться по ширине найденной линии или по типовой физической ширине подписи.
- Все размеры хранить в PDF points, но в настройках можно показывать миллиметры.

## 5. AI Layer

### 5.1 Конфигурация OpenAI

Использовать OpenAI API через переменные окружения:

```env
OPENAI_API_KEY=...
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=...
```

`OPENAI_MODEL` должен быть конфигурируемым. Нельзя жестко зашивать непроверенное имя модели вроде `gpt-5.4-mini`. Для production-режима модель нужно выбирать из доступных моделей аккаунта и актуальной документации OpenAI. Если нужна более дешевая модель, ее slug должен задаваться через `OPENAI_MODEL`.

Источник для проверки актуальных рекомендаций по моделям: [OpenAI latest model docs](https://developers.openai.com/api/docs/guides/latest-model).

### 5.2 Приватность

В UI обязательно нужна настройка `Использовать AI-анализ`.

Если она включена, пользователь должен понимать, что в OpenAI API отправляются подготовленные фрагменты документа:

- OCR-текст страницы;
- координаты слов;
- список найденных линий;
- кандидатные зоны подписи;
- небольшие изображения-фрагменты вокруг кандидатных зон.

Нельзя без необходимости отправлять весь документ целиком. Сначала нужно локально выделить релевантные страницы и области.

### 5.3 Что получает AI

AI получает структурированный контекст по каждой странице и каждому кандидату:

```json
{
  "document_id": "job_123",
  "page_number": 1,
  "page_size": {
    "width": 595.28,
    "height": 841.89
  },
  "ocr_text": "Фрагмент распознанного текста страницы",
  "words": [
    {
      "text": "Генеральный",
      "bbox": [80, 690, 160, 705]
    },
    {
      "text": "директор",
      "bbox": [165, 690, 230, 705]
    }
  ],
  "lines": [
    {
      "bbox": [240, 688, 420, 690],
      "width": 180,
      "type": "horizontal"
    }
  ],
  "candidates": [
    {
      "candidate_id": "target_1",
      "bbox": [230, 650, 450, 720],
      "anchor": "Генеральный директор",
      "local_confidence": 0.72,
      "reason": "line_near_role"
    }
  ]
}
```

Если используются изображения-фрагменты, они должны быть обрезками вокруг кандидатов, а не всей страницей, если только весь контекст страницы действительно не нужен.

### 5.4 Что возвращает AI

AI должен вернуть строго структурированный JSON:

```json
{
  "decisions": [
    {
      "candidate_id": "target_1",
      "should_sign": true,
      "should_stamp": true,
      "should_add_name": true,
      "name_text": "Венедиктов Р.В.",
      "signature_bbox": [238, 650, 420, 700],
      "stamp_bbox": [355, 635, 470, 750],
      "name_bbox": [425, 685, 520, 705],
      "confidence": 0.86,
      "needs_manual_review": false,
      "reason": "Найдена линия подписи рядом с ролью Генеральный директор; ФИО рядом отсутствует."
    }
  ]
}
```

Если AI не уверен, он обязан выставить `needs_manual_review: true`, а не принимать рискованное решение.

### 5.5 Случаи, которые должен различать AI

AI должен анализировать не только наличие фамилии, но и смысл зоны:

- `Венедиктов Р.В.` есть рядом с линией, и это место подписи.
- `Венедиктов Р.В.` есть в тексте договора, но это не зона подписи.
- Есть только `Генеральный директор`, и подпись нужна рядом с этой ролью.
- Есть несколько мест, где должен подписать один человек.
- Подпись нужна в одном месте, а печать - в другом.
- Есть строка `подпись`, но рядом указан другой человек.
- Есть линия без ФИО, но контекст показывает, что это подпись генерального директора.
- Документ уже содержит подпись или печать, и повторно вставлять не нужно.
- Надежного места нет, нужна ручная разметка.

### 5.6 Fallback без AI

Если OpenAI API недоступен, превышен лимит, не задан `OPENAI_API_KEY`, модель не найдена или произошла ошибка сети:

- сервис продолжает локальный OCR/CV-анализ;
- все спорные документы помечает как `needs_manual_review`;
- в UI показывает предупреждение;
- экспорт разрешает только после ручного подтверждения, если включена соответствующая настройка.

## 6. Core Algorithm

### 6.1 Подготовка PDF

Для каждого PDF:

1. Открыть документ.
2. Получить количество страниц.
3. Для каждой страницы определить размер в PDF points.
4. Извлечь текстовый слой, если он есть.
5. Если текстового слоя нет или он плохой, выполнить OCR.
6. Отрендерить страницы в изображения для CV-анализа и предпросмотра.

Рендерить страницы для анализа в высоком разрешении, например 200-300 DPI. Для UI можно использовать отдельные preview-изображения меньшего размера.

### 6.2 Извлечение текста

Порядок:

1. Сначала использовать PDF text layer.
2. Если текста мало или нет координат, использовать OCR.
3. Нормализовать текст:
   - привести к нижнему регистру для поиска;
   - заменить частые OCR-ошибки;
   - убрать лишние пробелы;
   - сохранить исходный текст для отображения.

Искать якоря:

- `Венедиктов`
- `Венедиктов Р.В.`
- `Венедиктов Р В`
- `Генеральный директор`
- `директор`
- `подпись`
- `ФИО`
- `расшифровка подписи`

Нужно поддержать fuzzy matching для OCR-ошибок. Например, фамилия может быть распознана с ошибкой в одной букве.

### 6.3 Поиск линий

Линии подписи искать двумя способами:

1. Через векторные объекты PDF, если линия сохранена как графика.
2. Через компьютерное зрение на отрендеренной странице.

Признаки линии подписи:

- горизонтальная;
- длина больше минимального порога;
- находится рядом с текстом роли, ФИО или словом `подпись`;
- вокруг есть свободное место;
- расположена в нижней или средней части блока подписания.

Линии таблиц и подчеркивания в обычном тексте нужно понижать в confidence.

### 6.4 Формирование кандидатов

Каждый кандидат `SignatureTarget` должен содержать:

- страницу;
- координаты линии;
- координаты ближайших слов;
- тип якоря;
- локальный confidence;
- причину выбора;
- предупреждения.

Примеры причин:

- `line_near_venediktov`
- `line_near_general_director`
- `empty_signature_line`
- `stamp_area_near_signature`
- `ambiguous_multiple_lines`
- `low_text_confidence`

### 6.5 Выбор финального размещения

Сначала локальный алгоритм формирует кандидатов, затем AI уточняет решение, если включен.

Правила размещения подписи:

- подпись ставить в область линии подписи;
- нижняя часть подписи может немного заходить на линию;
- подпись не должна закрывать ФИО полностью;
- если линия широкая, подпись масштабировать до 60-90% ширины линии;
- если линия узкая, использовать типовой физический размер и отправить документ на ручную проверку.

Правила размещения печати:

- печать ставить рядом с подписью;
- печать должна частично перекрывать подпись;
- перекрытие ориентировочно 15-35% площади подписи;
- печать не должна полностью закрывать читаемую фамилию;
- если в документе есть отдельная зона `М.П.` или `печать`, печать можно поставить туда, даже если это не совпадает с подписью;
- если AI считает, что подпись и печать должны быть в разных местах, создать два отдельных placement.

Правила вставки ФИО:

- вставлять `Венедиктов Р.В.` только если включена соответствующая галочка;
- не вставлять повторно, если ФИО уже есть рядом с местом подписи;
- если есть только `Венедиктов`, но нет инициалов, можно добавить `Р.В.` или полную строку рядом с зоной подписи;
- стиль текста должен быть близок к документу: размер, цвет, базовая линия.

## 7. Backend API

Минимальные endpoints:

### `POST /upload`

Загружает несколько PDF.

Возвращает:

```json
{
  "jobs": [
    {
      "job_id": "job_123",
      "filename": "document.pdf",
      "status": "uploaded"
    }
  ]
}
```

### `POST /analyze`

Запускает анализ документов.

Вход:

```json
{
  "job_ids": ["job_123"],
  "options": {
    "place_signature": true,
    "place_stamp": true,
    "add_name_if_missing": true,
    "use_ai": true,
    "require_manual_confirmation": true
  }
}
```

Возвращает список найденных placements и предупреждений.

### `GET /preview/{job_id}`

Возвращает страницы предпросмотра и текущие placements.

### `POST /placement/{job_id}`

Сохраняет ручные правки пользователя.

### `POST /export`

Генерирует PDF или ZIP.

### `GET /download/{export_id}`

Скачивает готовый файл.

- Извлекать текстовый слой и координаты слов.

- Найти горизонтальные линии.

- Сделать локальную веб-страницу загрузки PDF.
- Добавить чекбоксы настроек.
- Показать список документов и статусы.

### Этап 6. Экспорт

- Генерировать PDF с наложенными подписью, печатью и ФИО.

- Экспортировать отдельные PDF.
- Экспортировать ZIP для нескольких документов.

## 10. Обязательные тесты

### 10.1 PDF-анализ

- PDF с явной строкой `Венедиктов Р.В.` и линией: подпись ставится правильно.
- PDF, где `Венедиктов Р.В.` есть в тексте, но не в зоне подписи: подпись не ставится автоматически.
- PDF только с `Генеральный директор`: AI предлагает место подписи у этой роли.
- PDF без ФИО: при включенной галочке добавляется `Венедиктов Р.В.`.
- PDF с несколькими нужными местами: создается несколько placements.
- PDF, где подпись и печать должны быть в разных местах: placements разделяются.
- PDF без надежных якорей: документ получает `needs_manual_review`.

### 10.2 Опции пользователя

- Режим `только подпись`: печать не вставляется.
- Режим `только печать`: подпись не вставляется.
- Режим `подпись + печать`: печать частично перекрывает подпись.
- Отключенный `Добавить Венедиктов Р.В.`: ФИО не вставляется.
- Отключенный AI: используется только локальный алгоритм.
- Включенный `Требовать ручное подтверждение`: экспорт без подтверждения запрещен.

### 10.3 AI и fallback

- AI возвращает валидный JSON: решение применяется.
- AI возвращает невалидный JSON: используется fallback и предупреждение.
- OpenAI API недоступен: сервис продолжает локальный анализ.
- `OPENAI_API_KEY` не задан: AI отключается с понятным предупреждением.
- Низкий confidence AI: документ требует ручной проверки.

### 10.4 Предпросмотр

- Ручное перемещение подписи сохраняется в экспортированном PDF.
- Ручное перемещение печати сохраняется в экспортированном PDF.
- Изменение размера сохраняется.
- Удаленный placement не попадает в экспорт.
- Вручную добавленный placement попадает в экспорт.

### 10.5 Экспорт

- PNG-прозрачность подписи и печати сохраняется.
- Размер подписи и печати выглядит реалистично на A4.
- Печать частично перекрывает подпись, но не закрывает ФИО полностью.
- Пакетная обработка нескольких PDF работает независимо.
- Экспортированный PDF открывается в Preview и Adobe Reader.
- ZIP содержит все успешно обработанные документы.
- Исходные файлы не перезаписываются.

## 11. Acceptance Criteria

Первая версия считается готовой, если:

- можно загрузить несколько PDF;
- сервис находит хотя бы типовые зоны подписи рядом с `Венедиктов Р.В.` и `Генеральный директор`;
- можно выбрать подпись, печать и добавление ФИО отдельными галочками;
- AI-анализ можно включить и выключить;
- при ошибке AI приложение не падает;
- пользователь может вручную поправить размещение;
- экспортированный PDF визуально совпадает с предпросмотром;
- подпись и печать имеют реалистичный размер;
- печать корректно частично перекрывает подпись;
- можно скачать один PDF или ZIP;
- исходники остаются неизменными.

## 12. Допущения и ограничения v1

- Поддерживается только PDF.
- Приложение локальное, не публичный SaaS.
- Ручной предпросмотр обязателен.
- Все спорные документы требуют ручной проверки.
- Дефолтное ФИО: `Венедиктов Р.В.`
- Дефолтные изображения подписи и печати находятся в `/Users/rafailvv/Documents`.
- AI помогает выбрать место, но пользователь остается финальным контролером результата.
- DOCX, электронная подпись, юридическая ЭЦП и облачное хранение не входят в первую версию.
