# Тест-план AI-сервиса подписи PDF

Этот документ описывает конкретные тесты, которые нужно выполнить, чтобы проверить качество сервиса подписи PDF. Тесты разделены по уровням: unit, integration, AI/fallback, UI, export и ручная визуальная проверка.

Автоматизированные тесты находятся в `tests/` и запускаются командой:

```bash
./.venv/bin/python -m pytest -q
```

Ручная визуальная проверка реальных документов вынесена в `VISUAL_QA.md`.

Приоритеты:

- `P0` - обязательный тест перед любой рабочей сборкой.
- `P1` - обязательный тест перед первой качественной версией.
- `P2` - расширенный тест для повышения надежности.

## 2. Unit Tests

### UT-001. Валидация `DocumentJob`

Приоритет: `P0`

Шаги:

1. Создать `DocumentJob` с корректным `job_id`, `filename`, `source_path`, `status`.
2. Создать `DocumentJob` без `job_id`.
3. Создать `DocumentJob` с неизвестным статусом.

Ожидаемый результат:

- Корректный объект создается.
- Объект без `job_id` отклоняется.
- Неизвестный статус отклоняется.

### UT-002. Валидация координат bbox

Приоритет: `P0`

Шаги:

1. Передать bbox `[10, 20, 100, 80]`.
2. Передать bbox `[100, 80, 10, 20]`.
3. Передать bbox с отрицательными координатами.
4. Передать bbox за пределами страницы.

Ожидаемый результат:

- Валидный bbox принимается.
- Перевернутый bbox отклоняется.
- Отрицательные координаты отклоняются или нормализуются только по явному правилу.
- Bbox за пределами страницы помечается ошибкой.

### UT-003. Конвертация preview pixels в PDF points

Приоритет: `P0`

1. Задать страницу PDF `595x842`.
2. Задать preview `1190x1684`.

4. Конвертировать в PDF points и обратно.

Ожидаемый результат:

- Координаты после обратной конвертации совпадают с исходными с допустимой погрешностью.
- Масштабирование не меняет пропорции.

## 8. Regression Checklist перед релизом

Перед каждой рабочей сборкой проверить:

- `POST /upload` принимает несколько PDF.
- `POST /analyze` не падает без OpenAI key.
- `GET /preview/{job_id}` возвращает страницы и placements.
- Ручное перемещение подписи сохраняется.
- Ручное перемещение печати сохраняется.
- Экспорт одного PDF работает.
- ZIP-экспорт работает.
- Исходные PDF не изменяются.
- PNG-прозрачность сохраняется.
- Низкий confidence ведет к ручной проверке.
- Поврежденный PDF не ломает пакет.
- UI показывает warnings.
- Экспортированный PDF открывается в Preview/Adobe Reader.

## 9. Минимальный набор тестов для MVP

Минимум, без которого MVP нельзя считать рабочим:

1. `UT-003` - конвертация координат.
2. `UT-007` - расчет подписи.
3. `UT-008` - расчет печати.
4. `UT-009` - перекрытие печати и подписи.
5. `IT-001` - загрузка одного PDF.
6. `IT-002` - загрузка нескольких PDF.
7. `IT-004` - анализ PDF с `Венедиктов Р.В.`.
8. `IT-006` - анализ PDF только с `Генеральный директор`.
9. `AI-001` - AI выключен.
10. `AI-002` - нет `OPENAI_API_KEY`.
11. `UI-003` - предпросмотр PDF.
12. `UI-004` - ручное перемещение подписи.
13. `UI-005` - ручное перемещение печати.
14. `EX-001` - экспорт одного PDF.
15. `EX-002` - экспорт ZIP.
16. `EX-004` - PNG-прозрачность.
17. `EX-007` - подпись и печать вместе.
18. `VQA-001` - реалистичный размер подписи.
19. `VQA-002` - реалистичный размер печати.
20. `VQA-003` - корректное перекрытие печати и подписи.
